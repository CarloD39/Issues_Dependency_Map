GitLab Dependencies Map - Guida

Guida

Questo strumento permette di visualizzare le dipendenze tra issue GitLab sotto forma di grafo. È pensato per supportare planning, refinement e coordinamento tra team.

Tipi di mappa

Iteration Map

  • Mostra le issue appartenenti a una o più iteration
  • Ideale per analizzare uno sprint
  • Permette di individuare blocchi e dipendenze temporali

Epic Map

  • Mostra le issue appartenenti a una specifica epic
  • Le issue esterne collegate vengono mostrate in grigio
  • Ideale per comprendere lo scope di una feature

Filtri principali

Scope

  • All → tutte le issue
  • Assigned to me → solo issue assegnate a te
  • Created by me → issue create da te

Issues relation types

  • All → tutte le relazioni
  • Block/Blocked by → solo relazioni di blocco

Questo filtro è fondamentale per il planning: limita la vista alle dipendenze critiche.

Scheduling types

  • All → tutte le issue
  • Planned only → solo issue con label di pianificazione

Out of scope issues

  • Show → include issue esterne collegate
  • Hide → mostra solo issue nello scope selezionato

In Iteration Map → esterne = fuori iteration In Epic Map → esterne = fuori epic

Epic (Parent filter)

  • All → mostra tutte le epic
  • Open Epic → nasconde epic concluse (QA Passed)
  • None → nasconde tutte le epic

Come combinare i filtri

Analisi dei blocchi (consigliato)

  • Issues relation types → Block/Blocked by
  • Out of scope → Show

Permette di vedere cosa blocca lo sprint anche fuori iteration o epic.

Vista pulita

  • Out of scope → Hide

Mostra solo ciò che appartiene allo scope selezionato.

Focus sul lavoro pianificato

  • Scheduling types → Planned only

Come leggere la mappa

Nodi (issue)

  • Blu → issue nello scope selezionato
  • Grigio → issue esterna collegata
  • Bordo verde → issue chiusa

Epic / Parent

  • Rappresentate come rettangoli
  • La epic selezionata è evidenziata

Relazioni

  • Linea arancione → blocks / blocked_by
  • Linea tratteggiata → relates_to

Le relazioni mostrano come le issue dipendono tra loro.

Interazioni

  • Hover → mostra dettagli della issue
  • Doppio click → apre l’issue su GitLab
  • Scroll → zoom
  • Drag → sposta la mappa

Consigli pratici

  • Usa Block/Blocked by per il planning
  • Attiva Out of scope = Show per vedere dipendenze reali
  • Disattiva le epic se vuoi una vista più pulita
  • Usa la Epic Map per capire l’impatto di una feature
